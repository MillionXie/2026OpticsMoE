# OpticalMoE 每周进度小结

时间：2026-06-02  
主题：大画布光学 MoE 框架搭建与双专家任务切换验证

## 1. 本周主要进展

本周主要完成了一个大画布 OpticalMoE 原型系统。整体思路是：将不同任务对应的光学专家放置在同一块较大的光学仿真平面中，通过任务相关的光栅 prompt 将输入光场引导到指定专家区域，再由对应专家完成分类。

目前系统中已经包含两个专家：

- 左侧专家：用于 MNIST 手写数字识别
- 右侧专家：用于 FashionMNIST 服饰图像识别

也就是说，本周不仅完成了单个专家的训练和验证，也完成了多个专家在同一光学框架下的组合与任务切换验证。

## 2. 当前方案说明

当前采用的是“固定任务切换”的 OpticalMoE 方案。具体来说，输入图像先进入中心输入区域，然后根据当前任务选择不同的光栅相位：

- 当前任务为 MNIST 时，光场被引导到左侧专家
- 当前任务为 FashionMNIST 时，光场被引导到右侧专家

这个过程相当于使用一个低速可切换的光学 prompt 作为任务控制信号。任务切换时只需要切换 prompt，相同的输入光学主通路和大画布结构可以复用。

目前还没有训练真正的输入自适应 router，因此现阶段更接近“已知任务条件下的专家切换验证”。这个结果可以作为后续训练 optical prompt/router 的基础。

## 3. 实验效果

MNIST 样本被路由到左侧专家后，测试集分类准确率为：

|---|---|---:|
| MNIST | 左侧专家 | **89.68%** |

结果说明：左侧专家在大画布 OpticalMoE 框架下能够正常完成 MNIST 分类，说明输入光场能够被较稳定地引导到左侧专家区域。

## 4. 本周结论

本周已经验证了以下几点：

- 可以在同一个大画布光学系统中放置多个专家。
- 不同任务可以通过不同光栅 prompt 被引导到不同专家区域。
- 单个专家在迁入大画布后仍然可以保持可用的分类能力。
- 两个专家可以在同一系统中分别完成 MNIST 和 FashionMNIST 任务。
- 当前结果已经初步体现了“光学 prompt 控制专家选择”的 OpticalMoE 思路。

从结果看，MNIST 左专家达到 **89.68%**，FashionMNIST 右专家达到 **79.41%**。这说明当前结构具备基本可行性，但距离真正的自动路由 OpticalMoE 还有进一步工作。

## 5. 遇到的问题与思考

本周最主要的问题不是代码实现，而是光学结构设计本身存在较强的几何约束。

在设计光栅相位时，专家布局、传播距离、光栅周期、偏转角度和专家间距是相互耦合的。如果左右专家距离过近，光场容易在传播过程中发生串扰；如果距离过远，则需要更大的偏转角，可能带来采样误差、位置偏移和后续传播不稳定。

另外，光栅 prompt 产生的不是简单的“平移”，而是带有传播角度的偏转。因此光场到达专家入口后，还需要入口反向光栅进行 de-tilt，否则光场会在后续传播层中持续横向漂移。这个问题在多专家布局中比较关键，因为一旦入口位置或角度没有对齐，后续层的能量分布就会逐渐偏离专家区域。

本周也发现，如果先在单专家小画布中训练好模型，再直接迁移到大画布 MoE 框架中，性能可能会下降。原因是迁移后的输入波前、传播距离、光栅偏转角和专家入口位置都发生了变化，原来的专家并不一定天然适配新的光路。因此后续更合理的流程可能是：先确定大画布网格、专家位置和光栅几何，再在这个框架下训练或微调专家。

## 6. 目前采取的解决方式

针对上述问题，本周采用了以下处理方式：

- 先固定大画布尺寸和左右专家位置，保证整体几何框架稳定。
- 通过光栅 prompt 将中心输入分别引导到左、右专家。
- 在专家入口加入 de-tilt 相位，减少后续传播中的横向漂移。
- 将单专家模型迁入大画布后，再进行专家相位微调，使其适配新的大画布传播环境。
- 增加多类别样本可视化，用于观察不同类别在专家 detector 上的响应是否合理。

这些处理后，两个专家都能在大画布框架中完成对应任务。

## 7. 下一步计划

下一步计划主要有三个方向：

1. 继续优化专家布局和光栅参数，降低 outside energy 和专家间串扰。
2. 在当前“手动任务切换”的基础上，引入可训练 optical prompt 或 router，使系统可以根据任务或输入自动选择专家。
3. 尝试更多任务组合，验证当前 OpticalMoE 框架是否能够扩展到更多专家和更多任务。

整体来看，本周完成了 OpticalMoE 从单专家到双专家任务切换的第一版验证，为后续做真正的光学 MoE 路由训练打下了基础。
